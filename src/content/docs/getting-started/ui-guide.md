---
title: UI Guide
description: What each Tear HUD element communicates during a run.
---
Tear’s HUD is deliberately compact: every signal is gameplay information.

## Vitality

The health display shows current survival state and available shields. Taking damage also disrupts Style, so protecting health preserves both survival and offensive momentum.

## Style

The Style Meter records varied combat actions and shows the current rank. Higher ranks strengthen the run, while inactivity and damage reduce momentum. See [Style Meter](/mechanics/style-meter) for live thresholds.

## Run state

The wave readout, enemy queue, coin balance, and score cluster show what remains in the encounter and what the run has earned. Use the queue to prepare for the next threat instead of reacting late.

## Blade state

The reticle shows blade aim and communicates whether the blade is held, thrown, embedded, or returning. See [The Blade](/mechanics/the-blade) for live spring and throw behavior.

## Contrast

HUD colors adapt to the current biome so essential signals remain readable against either light or dark arenas.
